Many PHP developers need to send email from their code. The only PHP function that supports this directly is \href{https://www.php.net/manual/en/function.mail.php}{\texttt{ {\ttfamily mail()}}}. However, it does not provide any assistance for making use of popular features such as encryption, authentication, HTML messages, and attachments.

Formatting email correctly is surprisingly difficult. There are myriad overlapping (and conflicting) standards, requiring tight adherence to horribly complicated formatting and encoding rules – the vast majority of code that you\textquotesingle{}ll find online that uses the {\ttfamily mail()} function directly is just plain wrong, if not unsafe!

The PHP {\ttfamily mail()} function usually sends via a local mail server, typically fronted by a {\ttfamily sendmail} binary on Linux, BSD, and mac\+OS platforms, however, Windows usually doesn\textquotesingle{}t include a local mail server; \mbox{\hyperlink{namespace_p_h_p_mailer}{PHPMailer}}\textquotesingle{}s integrated SMTP client allows email sending on all platforms without needing a local mail server. Be aware though, that the {\ttfamily mail()} function should be avoided when possible; it\textquotesingle{}s both faster and \href{https://exploitbox.io/paper/Pwning-PHP-Mail-Function-For-Fun-And-RCE.html}{\texttt{ safer}} to use SMTP to localhost.
