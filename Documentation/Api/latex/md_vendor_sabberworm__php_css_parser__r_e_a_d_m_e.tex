\hypertarget{md_vendor_sabberworm__php_css_parser__r_e_a_d_m_e_autotoc_md73}{}\doxysubsection{Manipulation}\label{md_vendor_sabberworm__php_css_parser__r_e_a_d_m_e_autotoc_md73}
The resulting data structure consists mainly of five basic types\+: {\ttfamily CSSList}, {\ttfamily Rule\+Set}, {\ttfamily Rule}, {\ttfamily Selector} and {\ttfamily Value}. There are two additional types used\+: {\ttfamily Import} and {\ttfamily Charset} which you won’t use often.\hypertarget{md_vendor_sabberworm__php_css_parser__r_e_a_d_m_e_autotoc_md74}{}\doxysubsubsection{CSSList}\label{md_vendor_sabberworm__php_css_parser__r_e_a_d_m_e_autotoc_md74}
{\ttfamily CSSList} represents a generic CSS container, most likely containing declaration blocks (rule sets with a selector) but it may also contain at-\/rules, charset declarations, etc. {\ttfamily CSSList} has the following concrete subtypes\+:


\begin{DoxyItemize}
\item {\ttfamily Document} – representing the root of a CSS file.
\item {\ttfamily Media\+Query} – represents a subsection of a CSSList that only applies to a output device matching the contained media query.
\end{DoxyItemize}

To access the items stored in a {\ttfamily CSSList} – like the document you got back when calling {\ttfamily \$o\+Css\+Parser-\/$>$parse()} –, use {\ttfamily get\+Contents()}, then iterate over that collection and use instanceof to check whether you’re dealing with another {\ttfamily CSSList}, a {\ttfamily Rule\+Set}, a {\ttfamily Import} or a {\ttfamily Charset}.

To append a new item (selector, media query, etc.) to an existing {\ttfamily CSSList}, construct it using the constructor for this class and use the {\ttfamily append(\$o\+Item)} method.\hypertarget{md_vendor_sabberworm__php_css_parser__r_e_a_d_m_e_autotoc_md75}{}\doxysubsubsection{Rule\+Set}\label{md_vendor_sabberworm__php_css_parser__r_e_a_d_m_e_autotoc_md75}
{\ttfamily Rule\+Set} is a container for individual rules. The most common form of a rule set is one constrained by a selector. The following concrete subtypes exist\+:


\begin{DoxyItemize}
\item {\ttfamily At\+Rule\+Set} – for generic at-\/rules which do not match the ones specifically mentioned like @import, @charset or @media. A common example for this is @font-\/face.
\item {\ttfamily Declaration\+Block} – a Rule\+Set constrained by a {\ttfamily Selector}; contains an array of selector objects (comma-\/separated in the CSS) as well as the rules to be applied to the matching elements.
\end{DoxyItemize}

Note\+: A {\ttfamily CSSList} can contain other {\ttfamily CSSList}s (and {\ttfamily Import}s as well as a {\ttfamily Charset}) while a {\ttfamily Rule\+Set} can only contain {\ttfamily Rule}s.

If you want to manipulate a {\ttfamily Rule\+Set}, use the methods {\ttfamily add\+Rule(Rule \$o\+Rule)}, {\ttfamily get\+Rules()} and {\ttfamily remove\+Rule(\$m\+Rule)} (which accepts either a Rule instance or a rule name; optionally suffixed by a dash to remove all related rules).\hypertarget{md_vendor_sabberworm__php_css_parser__r_e_a_d_m_e_autotoc_md76}{}\doxysubsubsection{Rule}\label{md_vendor_sabberworm__php_css_parser__r_e_a_d_m_e_autotoc_md76}
{\ttfamily Rule}s just have a key (the rule) and a value. These values are all instances of a {\ttfamily Value}.\hypertarget{md_vendor_sabberworm__php_css_parser__r_e_a_d_m_e_autotoc_md77}{}\doxysubsubsection{Value}\label{md_vendor_sabberworm__php_css_parser__r_e_a_d_m_e_autotoc_md77}
{\ttfamily Value} is an abstract class that only defines the {\ttfamily render} method. The concrete subclasses for atomic value types are\+:


\begin{DoxyItemize}
\item {\ttfamily Size} – consists of a numeric {\ttfamily size} value and a unit.
\item {\ttfamily Color} – colors can be input in the form \#rrggbb, \#rgb or schema(val1, val2, …) but are always stored as an array of (\textquotesingle{}s\textquotesingle{} =$>$ val1, \textquotesingle{}c\textquotesingle{} =$>$ val2, \textquotesingle{}h\textquotesingle{} =$>$ val3, …) and output in the second form.
\item {\ttfamily CSSString} – this is just a wrapper for quoted strings to distinguish them from keywords; always output with double quotes.
\item {\ttfamily URL} – URLs in CSS; always output in URL(\char`\"{}\char`\"{}) notation.
\end{DoxyItemize}

There is another abstract subclass of {\ttfamily Value}, {\ttfamily Value\+List}. A {\ttfamily Value\+List} represents a lists of {\ttfamily Value}s, separated by some separation character (mostly {\ttfamily ,}, whitespace, or {\ttfamily /}). There are two types of {\ttfamily Value\+List}s\+:


\begin{DoxyItemize}
\item {\ttfamily Rule\+Value\+List} – The default type, used to represent all multi-\/valued rules like {\ttfamily font\+: bold 12px/3 Helvetica, Verdana, sans-\/serif;} (where the value would be a whitespace-\/separated list of the primitive value {\ttfamily bold}, a slash-\/separated list and a comma-\/separated list).
\item {\ttfamily CSSFunction} – A special kind of value that also contains a function name and where the values are the function’s arguments. Also handles equals-\/sign-\/separated argument lists like {\ttfamily filter\+: alpha(opacity=90);}.
\end{DoxyItemize}\hypertarget{md_vendor_sabberworm__php_css_parser__r_e_a_d_m_e_autotoc_md78}{}\doxysubsubsection{Convenience methods}\label{md_vendor_sabberworm__php_css_parser__r_e_a_d_m_e_autotoc_md78}
There are a few convenience methods on Document to ease finding, manipulating and deleting rules\+:


\begin{DoxyItemize}
\item {\ttfamily get\+All\+Declaration\+Blocks()} – does what it says; no matter how deeply nested your selectors are. Aliased as {\ttfamily get\+All\+Selectors()}.
\item {\ttfamily get\+All\+Rule\+Sets()} – does what it says; no matter how deeply nested your rule sets are.
\item {\ttfamily get\+All\+Values()} – finds all {\ttfamily Value} objects inside {\ttfamily Rule}s.
\end{DoxyItemize}\hypertarget{md_vendor_sabberworm__php_css_parser__r_e_a_d_m_e_autotoc_md79}{}\doxysection{To-\/\+Do}\label{md_vendor_sabberworm__php_css_parser__r_e_a_d_m_e_autotoc_md79}
